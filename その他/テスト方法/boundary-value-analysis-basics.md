## 境界値分析（Boundary Value Analysis） 基本例まとめ

### 3. 基本例①：数値入力（年齢）

**仕様**：年齢は **0〜120歳** まで入力可能

**テストデータ（境界値）**

- 下限の外：`-1` → エラー
- 下限ちょうど：`0` → OK
- 下限の内側：`1` → OK
- 上限の内側：`119` → OK
- 上限ちょうど：`120` → OK
- 上限の外：`121` → エラー

**ポイント**
- **0 と 120「ちょうど」** を必ず含める
- その **±1** を合わせて確認

---

### 4. 基本例②：文字列長（パスワード）

**仕様**：パスワードは **8文字以上 20文字以下**

**テストデータ（境界値）**

- 下限の外：`7文字` → エラー
- 下限ちょうど：`8文字` → OK
- 下限の内側：`9文字` → OK
- 上限の内側：`19文字` → OK
- 上限ちょうど：`20文字` → OK
- 上限の外：`21文字` → エラー

**実務あるある**
- 「8文字以上」のはずが **8文字が弾かれる**
- DBカラム制限不一致で **20文字保存時に失敗**

---

### 5. 基本例③：日付の境界

**仕様**：予約可能期間は **2026/04/01〜2026/04/30**

**テストデータ（境界値）**

- 下限の外：`2026/03/31` → エラー
- 下限ちょうど：`2026/04/01` → OK
- 上限ちょうど：`2026/04/30` → OK
- 上限の外：`2026/05/01` → エラー

**注意点**
- 月末の違い（30日/31日/2月）
- うるう年（2/29）の扱い

## 複数条件網羅（Multiple Condition Coverage） 基本例まとめ

### 3. 基本例①：2条件のAND

#### 仕様

```
条件A：会員である
条件B：有効期限内である

if (A and B) then 利用可能
else 利用不可
```

#### 条件組み合わせ

| No | A（会員） | B（期限内） | 結果 |
|---|---|---|---|
| 1 | True | True | 利用可能 |
| 2 | True | False | 利用不可 |
| 3 | False | True | 利用不可 |
| 4 | False | False | 利用不可 |

✅ **AND条件は1パターンしかTrueにならない**点が重要。

---

### 4. 基本例②：2条件のOR

#### 仕様

```
条件A：管理者である
条件B：承認者である

if (A or B) then 操作可能
else 操作不可
```

#### 条件組み合わせ

| No | A（管理者） | B（承認者） | 結果 |
|---|---|---|---|
| 1 | True | True | 操作可能 |
| 2 | True | False | 操作可能 |
| 3 | False | True | 操作可能 |
| 4 | False | False | 操作不可 |

✅ **OR条件はFalse/FalseだけがNG**。

---

### 5. 基本例③：3条件の組み合わせ

#### 仕様

```
if (A and B and C) then 処理実行
```

#### 組み合わせ数

- 条件数：3
- 組み合わせ：2³ = **8通り**

| No | A | B | C | 結果 |
|---|---|---|---|---|
| 1 | T | T | T | 実行 |
| 2 | T | T | F | 非実行 |
| 3 | T | F | T | 非実行 |
| 4 | T | F | F | 非実行 |
| 5 | F | T | T | 非実行 |
| 6 | F | T | F | 非実行 |
| 7 | F | F | T | 非実行 |
| 8 | F | F | F | 非実行 |

⚠ 条件が1つ増えるごとに、**ケース数は倍増**します。
